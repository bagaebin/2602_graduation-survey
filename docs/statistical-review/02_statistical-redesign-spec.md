# 통계 개선 설계 스펙 (Statistical Redesign Spec)

작성일: 2026-03-04  
적용 대상: `app.js`, `scripts/build_survey_data.py`, `survey-data.js`

## 1) 목적

본 스펙은 대시보드의 시각적 완성도와 별개로, 통계적 해석 타당성을 보장하기 위한 집계/표현 규칙을 정의한다.

## 2) 설계 원칙

1. 응답자 단위 기여도 균형
- 응답자마다 점수 총량이 과도하게 달라지지 않도록 정규화한다.

2. 소표본 보호
- 소표본 셀의 결과를 “강한 결론”으로 제시하지 않는다.

3. 불확실성 가시화
- 점추정치(평균/점수)만이 아니라 구간(오차)을 함께 제시한다.

4. 척도 일관성
- 문항의 실제 응답 패턴(동순위/부분응답)을 반영한 점수화 체계를 사용한다.

## 3) 데이터 계층 및 산출물

## 3.1 신규 산출물

1. `analysis-core.json`
- 차트 렌더 이전 단계에서 계산된 정규화 점수, 신뢰구간, 경고 플래그 저장

2. `quality-report.json`
- 결측률, 동순위율, 소표본 경고 통계 저장

## 3.2 기존 산출물 변경

- `survey-data.js`는 원자료 성격 유지
- 집계/추론 로직은 `analysis-core` 레이어로 분리

## 4) 점수화 규칙(필수)

## 4.1 순위형 문항(전시 방식/기대/우려)

현재: 단순 가산(1=3,2=2,3=1,결측=0)

개선: 응답자 정규화 tie-aware 점수

정의:

- 기본 가중치: `w(1)=3`, `w(2)=2`, `w(3)=1`, 그 외 0
- 응답자 `i`, 항목 `j`의 원점수 `s_ij = w(rank_ij)`
- 응답자 총점 `S_i = sum_j s_ij`
- 정규화 점수 `p_ij = s_ij / S_i` (단, `S_i>0`)

결과:

- 각 응답자는 해당 문항에서 총 1점만 기여
- 동순위/부분응답에서도 기여도 과대 문제가 줄어듦

## 4.2 Likert 문항(1~5)

현재: 평균만 표시

개선:

- 기본지표: 평균
- 보조지표: 중앙값, IQR, 응답분포(1~5 비율)

## 4.3 복수선택 예산 문항

- 항목별 비율 = 선택자 수 / 유효응답자 수
- 복수 선택 발생 시 다중응답으로 명시
- 단일선택 문항에서 복수 선택이 나오면 데이터 품질 경고 플래그 생성

## 5) 결측/품질 규칙

## 5.1 결측 처리

- 순위형: 결측은 0점 처리하되, 정규화 분모(`S_i`)에는 비결측만 사용
- 평균형: 리스트와이즈 삭제(해당 항목 결측만 제외)

## 5.2 품질 경고 플래그

필수 플래그:

- `warning_sparse_cell`: 집단 셀 `n < 5`
- `warning_high_missing`: 문항 응답률 `< 70%`
- `warning_tie_heavy`: 동순위 응답 비율 `> 30%`
- `warning_multi_in_single`: 단일선택 문항에서 복수 선택 감지

## 6) 불확실성(신뢰구간) 규격

## 6.1 순위형 지표

- 방식: 부트스트랩(재표본추출)
- 반복수: 기본 `B=2000` (최종 보고 `B=5000` 권장)
- 표기: 중앙값 + 95% 구간

## 6.2 평균형 지표

- 표기: 평균 + 95% CI
- 소표본(`n<10`)은 t-근사 또는 부트스트랩 우선

## 6.3 차트 반영 규칙

- KPI: 점추정 + `±` 구간 또는 괄호형 CI
- 막대차트: 에러바(가능 시)
- 히트맵: 값 옆에 `n` 노출 또는 툴팁에 `n, CI` 표시

## 7) 소표본 정책

1. `n < 3`: 값 숨김(마스킹) 또는 “표본 부족” 표기
2. `3 <= n < 5`: 값 표시 가능, 단 경고 아이콘/문구 필수
3. `n >= 5`: 일반 표시

## 8) 시각화 정책(통계 우선)

1. 색상 인코딩
- 현재의 상대스케일(최소=보라, 중간=버터, 최대=빨강) 유지 가능
- 단, 범례에 “현재 필터 내부 상대값 기준” 명시

2. 차트 간 비교
- 절대 비교 필요 화면에서는 고정 스케일 색상 사용
- 상대 스케일과 절대 스케일을 혼용할 경우 라벨로 명확히 분리

3. 분포 시각화
- 평균 기반 차트에는 분포 보조 차트를 병기

## 9) 구현 명세

## 9.1 파일 구조 제안

- `scripts/build_analysis_core.py`
  - 정규화 점수, CI, 품질 플래그 계산
- `analysis-core.js`
  - 브라우저에서 사용 가능한 전처리 결과
- `app.js`
  - 렌더만 담당(집계 최소화)

## 9.2 함수 명세(예시)

1. `compute_rank_normalized_scores(rows, field)`
- 입력: 응답 배열, 순위 필드명
- 출력: 항목별 점수, 항목별 점유율, 응답자 총점 분포

2. `bootstrap_rank_share(rows, field, B=2000)`
- 출력: 항목별 중앙값/CI

3. `build_quality_flags(rows)`
- 출력: 전역/문항/집단 단위 경고 플래그

## 10) QA 및 수용 기준(DoD)

## 10.1 통계 정확성

- [ ] 원점수/정규화 점수 수동 검산 샘플 5건 일치
- [ ] 부트스트랩 CI 재현 가능(시드 고정 테스트)
- [ ] 소표본 정책 분기 테스트 통과

## 10.2 해석 안정성

- [ ] 필터 변경 시 경고 플래그 동기화
- [ ] KPI와 차트의 점수 체계 불일치 없음
- [ ] 분포와 평균의 해석 충돌 시 경고 문구 표시

## 11) 단계별 실행 계획

### Phase 1 (즉시, 1~2일)
- 순위형 정규화 점수 도입
- 소표본/결측 경고 배지 도입
- 인사이트 문장에 경고 반영

### Phase 2 (단기, 2~4일)
- 부트스트랩 CI 추가
- Likert 분포 차트 추가
- 품질 리포트(JSON) 자동 생성

### Phase 3 (중기, 차기 설문 전)
- 설문 문항 재설계(순위 중복 허용 여부 명확화)
- 배포용 익명화 파이프라인 추가

## 12) 기대 효과

- 표본 크기와 응답 구조를 고려한 안전한 결론 도출
- 회의 중 과해석/오판 위험 감소
- 데이터 업데이트 시에도 일관된 통계 품질 유지
