# 그래프/통계 타당성 감사 보고서

작성일: 2026-03-04  
대상 코드: `app.js`, `scripts/build_survey_data.py`, `survey-data.js`

## 1) 감사 목적과 범위

본 보고서는 다음을 검토한다.

- 현재 그래프/지표가 통계적으로 어떤 의미를 갖는지
- 현재 점수화/집계 방식이 어떤 편향을 유발하는지
- 소표본/결측/동순위 응답에서 해석 위험이 어느 정도인지

## 2) 데이터 상태 요약 (근거 수치)

표본 수는 `N=26`이다.

### 2.1 표본 불균형

- 과정: 예술사 23명(88.5%), 전문사 3명(11.5%)
- 세부전공: 최소 3명, 최대 8명 (`n<5` 카테고리 2/5)
- 스튜디오: 최소 1명, 최대 7명 (`n<5` 카테고리 3/6)

해석: 분류 필터를 적용할수록 셀 표본이 급격히 줄며, 평균/순위 차트의 오차가 매우 커진다.

### 2.2 순위형 문항의 결측/부분응답

- 전시 방식 순위(`mode_ranking`): 평균 유효항목 3.35/4, 최소 3, 최대 4
- 기대 요인 순위(`expectation_ranking`): 평균 4.92/8, 최소 3, 최대 8
- 우려 요인 순위(`concern_ranking`): 평균 5.23/9, 최소 1, 최대 9

해석: 완전 순위보다 부분 순위 응답이 지배적이며, 결측 처리 규칙이 결과에 크게 영향한다.

### 2.3 동순위(같은 순위 중복) 응답 빈도

- 전시 방식: 9/26명(34.6%)
- 기대 요인: 11/26명(42.3%)
- 우려 요인: 12/26명(46.2%)

최대 동일순위 반복은 기대/우려에서 6회까지 관측됨.

해석: 현재 데이터는 “엄격한 순위(서열)”보다 “부분순위+동순위 허용” 구조에 가깝다.

### 2.4 예시 불확실성 (부트스트랩)

전시 방식 가중점수 비중(중앙값, 95% 구간):

- 온·오프라인 병행: 30.6% (25.5~35.6)
- 오프라인: 26.0% (20.2~32.0)
- 온라인: 22.0% (17.0~27.2)
- 전시 없음: 21.3% (14.6~28.0)

해석: 순위는 대체로 유지되지만 구간 중첩이 커서 “확정적 우열”로 단정하기 어렵다.

## 3) 핵심 감사 결과 (우선순위 기준)

### [P0] 순위형 문항에서 결측과 동순위를 단순 0/합산 처리하여 비교 타당성이 약함

근거:

- `rankToWeight(None)=0`로 결측을 자동 0점 처리 (`app.js`)
- 동순위가 빈번한데(최대 6개 동순위), 현재는 단순 가산

영향:

- 응답자마다 총점 규모가 달라져 “많이 체크한 사람”의 영향력이 커짐
- 부분응답/동순위 응답이 많은 그룹일수록 비교가 왜곡될 수 있음

권고:

- 응답자별 정규화 점수(합=1) 체계 도입
- 동순위는 분할 배분(equal split) 또는 tie-aware Borda 적용

### [P0] 소표본 분할 비교에 불확실성/경고 체계가 없음

근거:

- 스튜디오 최소 표본 1명, 전공 최소 3명
- 현재 차트는 신뢰구간/표본수 경고 없이 평균/합계만 표시

영향:

- 우연 변동이 구조적 차이처럼 보일 수 있음
- 의사결정 회의에서 과해석 위험

권고:

- 셀 표본 `n<5` 시 회색 처리 또는 경고 라벨
- 핵심 지표에 부트스트랩 95% 구간 제공

### [P1] Likert(1~5) 문항을 평균만 제시하여 분포 정보 손실

근거:

- 오프라인 입장/운영 관련 문항은 모두 평균 중심 표현

영향:

- 양극화(찬반 공존) 여부가 평균에 가려짐

권고:

- 평균 + 중앙값 + 응답분포(100% 누적 막대) 병기

### [P1] 기대/우려 문항의 측정척도 해석이 불명확

근거:

- 반복 순위 선택이 많아 “Top-3 랭킹”이라기보다 “중복 우선도 표기”에 가까움

영향:

- 현재 가중합은 척도 정의와 실제 응답 행태가 불일치할 수 있음

권고:

- 문항 해석을 “선호 강도 투표”로 명시하거나
- 차기 설문에서 순위 중복 불가 제약 적용

### [P2] 색상 스케일이 차트 내부 상대값 기준이라 차트 간 비교 오해 가능

근거:

- 최소/중간/최대 색은 각 차트의 로컬 범위에서 재스케일됨

영향:

- 같은 색이라도 차트 간 절대값 의미가 다름

권고:

- 색상에 “로컬 상대 스케일” 명시
- 절대 비교가 필요한 차트는 고정 스케일 사용

### [P2] 개인정보 포함 리스크

근거:

- `survey-data.js`에 `name` 필드 유지

영향:

- 대시보드 배포 범위가 넓어질 경우 개인정보 노출 가능

권고:

- 배포용 빌드에서 식별자 제거(익명화 모드)

## 4) 현재 결과 해석 시 안전 가이드

아래 원칙을 만족할 때만 강한 결론을 사용한다.

1. 집단 셀 표본이 5명 이상
2. 상위 대안의 신뢰구간이 하위 대안과 충분히 분리
3. 가중점수와 1순위 지표가 같은 방향
4. 평균 지표는 분포(응답 구성)와 함께 해석

## 5) 개선 우선순위 제안

1. P0 즉시: 순위형 점수 정규화 + 동순위 처리 + 소표본 경고
2. P1 단기: 분포형 차트 추가 + 불확실성 표기
3. P2 중기: 척도 정의 재정의 및 차기 설문 문항 개선

## 6) 결론

현재 대시보드는 “빠른 탐색”에는 유용하지만, 엄밀한 통계 해석 기준에서는 결측/동순위/소표본 처리 규칙이 미흡하다. 따라서 의사결정 문서로 직접 사용하기 전에, 최소한 P0 항목을 반영한 재집계 체계가 필요하다.
