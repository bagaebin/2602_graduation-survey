# 졸업전시 설문 반응형 리포트

이 저장소는 `survey.xlsx`(첨부 설문 원본)를 필터 가능한 웹 리포트로 변환합니다.

## 포함된 결과물

- `index.html`: 반응형 대시보드 화면
- `styles.css`: 레이아웃/테마 스타일
- `app.js`: 필터링 + Plotly 그래프 렌더링 로직
- `scripts/build_survey_data.py`: XLSX -> `survey-data.js` 데이터 변환 스크립트
- `survey-data.js`: 브라우저에서 바로 읽는 정규화 데이터

## 기획: 어떤 분류와 양상으로 볼까

아래 3개 축을 기본 분류로 두고 모든 그래프를 교차 확인하는 구성을 권장합니다.

1. 응답자 분류 축
- 과정: `예술사` vs `전문사`
- 세부전공: 인터랙션/커뮤니케이션/제품/운송/융합
- 스튜디오 분반(지도교수 기준)

2. 의사결정 축
- 전시 방식 선호: 오프라인/온라인/병행/전시 없음의 우선순위
- 비용 수용도: 개인 부담 적정 금액, 총 부담 가능 금액
- 운영 실행도: 일반 위원 참여 의사, 외부 인력 수용, 인건비 지급 수용

3. 의미/리스크 축
- 기대 요인: 진로 활용, 실무 경험, 네트워킹 등
- 우려 요인: 예산 부족, 공간 접근성, 조율 난이도, 참여 저조 등

### 실제 해석 포인트

- 합의 가능성: 전시 방식 1순위가 분류별로 같은지/갈리는지
- 실행 가능성: 선호 방식과 비용 수용 범위가 충돌하는지
- 운영 리스크: 참여 의사(인력)와 우려 요인(갈등/방임)의 동시 발생 여부
- 커뮤니케이션 우선순위: 기대 상위 항목과 우려 상위 항목의 간극

## 실행 방법

1. 데이터 다시 생성(원본 엑셀 갱신 시)

```bash
python3 scripts/build_survey_data.py --input survey.xlsx --output survey-data.js
```

2. 로컬 서버 실행

```bash
python3 -m http.server 8000
```

3. 브라우저 열기

- `http://localhost:8000`

## 확장 아이디어

- 텍스트 의견 자동 태깅(예: 비용/공정성/접근성) 추가
- 분류 조합(전공+스튜디오) 교차 히트맵 추가
- PDF 보고서 내보내기(회의 자료용)
